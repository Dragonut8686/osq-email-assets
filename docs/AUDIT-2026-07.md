## 3. Публичный GitHub как CDN — риски и альтернативы 💡

Сейчас: публичный репо → jsDelivr. Бесплатно и просто, но: (а) письма и акции
видны до отправки всем, (б) клиентские материалы утекали (п. 10), (в) лимиты
jsDelivr (файл ≤20 МБ, капризы на больших репо).

Варианты перехода (по возрастанию усилий):

1. **Оставить как есть + гигиена** (уже сделано): ref/ не коммитится, тяжёлое
   не попадает, purge автоматом. Риск (а) остаётся.
2. **Два репозитория**: рабочий приватный (всё), публичный `osq-email-cdn`
   (только index.html + images + shared, пушится скриптом на финале). Закрывает
   (а) и (б) почти полностью, CDN-ссылки меняются один раз.
3. **Cloudflare R2 + свой домен** (бесплатно до 10 ГБ, без лимитов jsDelivr,
   мгновенная инвалидация): репо становится полностью приватным, деплой через
   rclone/wrangler. Самый чистый вариант, ссылки вида `https://cdn.osqgroup.ru/…`
   выглядят солиднее для почтовых фильтров.

Рекомендую вариант 3 (или 2 как быстрый шаг). Готов реализовать по команде.
