## 3. Публичный GitHub как CDN — альтернативы, работающие в России 💡

Сейчас: публичный репо → jsDelivr. Бесплатно и просто, но: (а) письма и акции
видны всем до отправки, (б) лимиты jsDelivr (файл ≤20 МБ), (в) зависимость от
зарубежной инфраструктуры. Cloudflare R2 отпадает — не работает в РФ.

Важное ограничение любого переезда: ссылки в УЖЕ отправленных письмах заморожены
навсегда — публичный репо (или его копия) должен жить вечно, чтобы старые письма
не «ослепли». Поэтому переезд касается только НОВЫХ кампаний.

Варианты (все работают в РФ):

1. **Двухрепозиторная схема (GitHub, бесплатно)** — рабочий репо становится
   приватным (материалы, скиллы, скрипты), финальные ассеты кампании скрипт
   пушит в отдельный публичный `osq-email-cdn` (только index.html + images +
   shared). jsDelivr остаётся CDN. Усилия: ~час на скрипт, ссылки для новых
   писем меняются один раз. Закрывает риск (а) полностью.
2. **Yandex Cloud Object Storage + CDN** — S3-совместимое хранилище в РФ
   (~2–3 ₽/ГБ/мес, CDN ~1 ₽/ГБ трафика; при наших объёмах — десятки рублей
   в месяц). Свой домен `cdn.osqgroup.ru` — солиднее для почтовых фильтров
   и надёжнее для ру-получателей. Деплой через S3 API (скрипт готов написать).
   Рабочий репо приватный, jsDelivr остаётся только для старых писем.
3. **Selectel / VK Cloud / Timeweb S3 + CDN** — аналог варианта 2, тоже РФ;
   выбор — вопрос цены и удобства кабинета.

Рекомендация: **вариант 1 сейчас** (быстро и бесплатно закрывает приватность),
**вариант 2 как целевой** — когда захочется свой домен и полную независимость
от jsDelivr. Оба реализую по команде; старые ссылки не пострадают.
